DOCUMENTATION

Overview

Termux Launcher replaces your Android home screen with a live Termux session. Everything you'd tap for on a normal launcher — opening apps, searching, media, system status — happens in and around the prompt. It's built on Termux + Termux:Monet and inspired by TEL.

Requirements

  • A modern Android device (Termux-compatible).
  • The com.termux build cannot coexist with the regular Termux app — they share a package identity.
  • Recommended companion: the Unexpected Keyboard, which this launcher is tuned for.

Companion forks

Add-ons must be signed to match the launcher. Mixing differently-signed Termux add-ons causes shared-UID errors — always use the matching Termux:API and Termux:Styling forks from the downloads section.

The launcher surface

The terminal is the home screen. Launcher controls live around the Termux session.

Apps row

Long-press an icon to pin, move, or drop it in a folder. Long-press empty space for list-based management.

A–Z row

Swipe across to filter installed apps by letter. Swipe upward from a letter to launch an app in that group.

Terminal search

Type the split character before a query to search apps. Default is % — so %whatsapp finds WhatsApp.

Lock screen

Double-tap the alphabet row to lock the phone — once a lock method is configured in Apps Bar settings.

Long press Terminal → More → Appearance · Apps Bar · TAI
DOCUMENTATION

launcherctl bridge

launcherctl is a localhost API bridge that exposes Android and launcher data to shell tools without high-frequency polling. It's installed automatically when the launcher session starts, and authenticates with a bearer token in ~/.launcherctl/.

Try it first

$ launcherctl status
$ launcherctl apps
$ launcherctl launch whatsapp

The toolbox

resources · media · notifications · capabilities · tools · agent · events · mcp · update-scripts · token rotate. Media & notification commands need notification-listener access granted to the launcher.

Bind apps to tmux keys

Here Alt + w opens WhatsApp:

bind -n M-w run-shell 'tmux display-message "Opening WhatsApp"; \
  launcherctl launch whatsapp >/dev/null 2>&1 \
  || tmux display-message "Launch failed"'

Shizuku, rish & btop

Shizuku unlocks lock-screen control, a privileged shell, and the btop helper. Install and start Shizuku, then set up rish.

Set up rish

Let Shizuku create rish + rish_shizuku.dex, copy both into a directory on your $PATH, and set RISH_APPLICATION_ID="com.termux" at the bottom of rish. Then verify:

$ launcherctl tty-doctor
$ rish -c "id"

Then run the shell-setup script and choose All or btop only. On Android 14+, keep rish_shizuku.dex non-writable.

DOCUMENTATION

Agent & MCP

LauncherCtl exposes a shared tool registry for local agents, MCP clients, and scripts. Routing is deterministic — it does not load a hidden model. Execution is confirmation-gated for medium/high/critical risk tools.

Route & execute

$ launcherctl agent --dry-run "open maps"
$ launcherctl agent "open maps"

MCP stdio bridge

For MCP-capable clients (Codex, OpenCode, Crush). Requires Python in Termux.

$ pkg install python && launcherctl mcp

Troubleshooting

When the terminal feels stale or unresponsive, start here:

$ termux-reload-settings
$ launcherctl restart

Common fixes

401 Unauthorized — run launcherctl token rotate.
Connection refused — reopen the launcher, check ~/.launcherctl/endpoint.
No tty detected — run launcherctl tty-doctor.
media / notifications empty — grant notification access in Android settings.

$ tai download MODEL_ID URL --accept-terms
$ tai preflight MODEL_ID

$ tai keep-warm MODEL_ID --minutes 30

$ tai unload

Prefix any command with tai --json for raw API JSON you can pipe into jq and scripts. tai load, preflight, and keep-warm default to the assistant model when no ID is given. See tai --help for the full command set.

API REFERENCE

Introduction

Use TAI's local API to run and interact with models. The same server also powers launcherctl. Every route requires the bearer token.

Base URL & auth

TAI stores its current address and secret token in ~/.launcherctl/. OpenAI-compatible clients append /v1; Ollama clients use the base address as-is.

BASE=$(sed -n '1p' ~/.launcherctl/endpoint)
TOKEN=$(cat ~/.launcherctl/token)
export OPENAI_BASE_URL="$BASE/v1"
export OPENAI_API_KEY="$TOKEN"

Default bind mode is localhost (127.0.0.1). LAN mode is opt-in; treat the token as a network secret when enabled. No CORS headers are emitted — browser clients can't call the API directly.
